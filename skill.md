## 2. 디자인 규칙 (절대 기준)

### 색상
- **배경**: 흰색 `FFFFFF` 고정
- **메인**: 네이비 `1F3864`
- **보조**: 회색 `D9D9D9`(헤더박스) · `F2F2F2`(키워드/카드박스) · 연블루 `DEEAF6`/`DEEBF7`
- **액센트**: 블루 `376092` (RGB 55·96·146 — 포인트로만 절제하여 사용).
  숫자·점·번호 배지·아이콘 원 등 강조 포인트에 사용한다. 메인 네이비보다 밝아 위계가 형성된다.
- **금지**: **녹색/틸 계열 사용 금지**(이모지·도형·아이콘·액센트 전반),
  토픽과 무관한 기본 블루 남용, cream/beige 배경(`F5F5DC` 등)

### 표지 (커버)
- **배경은 흰색 `FFFFFF` 고정.** 다크 네이비 등 어두운 배경을 사용하지 않는다.
- **보고서 제목: 20pt, 검은색 `000000`, 가운데 정렬**
- **제목 바로 아래 회색 실선**(약 2pt, 회색 `808080`~`A6A6A6`)을 가운데 배치하되,
  **실선 길이는 제목 글자 폭보다 양옆으로 길어야 한다(예외 없는 절대 규칙).**
  제목 길이가 변하면 실선 길이도 항상 제목보다 길게 다시 설정한다. 제목과 실선 간격은 좁게 유지한다.
- **우측 상단 모서리에 빨간 박스**(가로 약 0.95"/2.4cm, 세로 약 0.32"/0.8cm, 빨강 `FF0000`,
  테두리 흰색 `FFFFFF`)에 "대외비"를 흰색 12pt **bold**로 표기한다(보안 등급).
  **이 대외비 박스는 커버를 포함한 모든 본문 슬라이드의 우상단에 동일하게 필수 표기한다.**
- **제목 외 부제·설명·태그라인 등은 표기하지 않는다**(제목 + 조직명만).
  단, "과제추진계획서"와 같은 문서 종류 라벨은 제목 아래 부제로 1줄 허용한다.
- **우측 하단에 조직명을 표기한다: 14pt, 검은색 `000000`, 오른쪽 정렬**(푸터 영역 우측).
- **장식 도형(원형 모티프 등)은 배치하지 않는다.** 제목·실선·조직명만으로 정돈한다.
- **조직 로고**: 이미지 파일이 제공된 경우에만 좌상단에 배치한다(가로형 로고 폭 약 2",
  비율 유지). 로고 이미지가 없으면 **회사명을 텍스트로 임의 생성하지 않는다**(빈 상태로 둔다).

### 폰트 (맑은 고딕, 영문 포함 전체)
| 요소 | 크기 | 색·굵기 |
|------|------|------|
| 타이틀 | **16pt 고정** (자동조정·확대 금지) | 검은색 `000000`, bold |
| 거버닝 메시지 | **14pt 고정** | 검은색 `000000`, bold |
| 본문 | 12pt (밀도 따라 9~12) | — |
| 캡션·출처 | 8~9pt | 회색 |
| 큰 숫자 콜아웃 | 28~32pt (과도한 확대 금지) | 네이비 |
| 커버 제목 | **20pt** | 검은색 `000000`, bold, 가운데 |
| 조직명(커버) | **14pt** | 검은색 `000000`, bold |
| 대외비 박스 | 12pt, **bold** | 흰색 `FFFFFF` |

> **거버닝 폰트 조정(v3):** 거버닝을 14pt로 정한다. 타이틀(16pt)과 위계가 구분되고,
> 14pt bold 2줄이 본문 시작선(y≈1.7") 위 영역에 안정적으로 수용되어
> "16pt 고정 + 2줄 + 본문 시작 y≈1.7"의 공간 충돌이 해소된다.

### 그리드 (절대 이탈 금지)
```
left = 0.5"   right = 9.5"   headerBot ≈ 1.45"   footerTop = 5.2"
```
모든 요소는 이 범위 안에 배치한다. **y + height ≤ 5.25"**, **x 범위 0.45"~9.55"**를 보장한다.
특히 우측에 요소가 여러 개 쌓이는 레이아웃(차트+설명)은 하단 박스가 footerTop을 초과하기 쉬우므로,
간격을 계산하여 경계 안에 배치한다. 16:9 = 10" × 5.625".

**[2단 분할 경계 규칙 — v3 신규, 좌/우 컬럼 겹침 방지]**
좌·우 2단 구성 시 두 블록의 좌표 경계를 다음과 같이 고정한다.
```
좌블록 영역 : x = 0.5"  ~  5.0"   (좌블록 모든 요소의 우측 끝 ≤ 5.0")
거터(중앙 여백) : 5.0" ~ 5.15"     (폭 0.15", 어떤 요소도 배치하지 않음)
우블록 영역 : x = 5.15" ~ 9.5"     (우블록 모든 요소의 좌측 시작 ≥ 5.15")
```
- **좌블록의 표·도형·차트는 우측 끝이 5.0"를 넘지 않도록 폭을 계산한다.**
- **우블록의 카드·텍스트박스는 좌측 시작 x가 5.15" 이상이어야 한다.**
- 좌블록 우측 끝(≤5.0)과 우블록 좌측 시작(≥5.15) 사이 거터(0.15")는 항상 비워 둔다.

**[우측 여백 금지]**
가로로 펼쳐지는 요소(표·이미지·도형·차트)는 콘텐츠 폭(좌 0.5"~우 9.5")을 기준으로 정렬한다.
정렬되지 않은 빈 우측 여백을 남기지 않는다.

**슬라이드 크기**: 기본은 와이드스크린 16:9 (10"×5.625").
별도 요청 시에만 표준 4:3으로 생성한다.

### 카드
- **둥근 모서리** `roundRect` (rectRadius 0.08) + **은은한 그림자**
  (`shadow:{type:"outer",blur:6,offset:2,angle:90,color:"BFBFBF",opacity:0.4}`)
- 테두리는 옅게(`E8E8E8` 0.5pt) 또는 생략

### 타이틀 / 거버닝 메시지
- 타이틀: 좌상단, 16pt bold, **검은색 `000000`**, **번호 없음**
- **타이틀과 거버닝 사이에 가는 회색 실선**(약 1pt, 회색 `808080`)을 배치한다.
- 거버닝: 회색 실선 바로 아래, **14pt, bold, 검은색 `000000`**. 해당 슬라이드의 실제 전달 메시지.
- **타이틀·실선·거버닝은 슬라이드 상단에 밀착 배치한다**(상단 여백 최소화).
- **거버닝과 본문 콘텐츠 사이에 충분한 세로 간격을 둔다**(본문 시작 y ≈ 1.7" 이상 권장).
- 거버닝 작성법:
  - **기본값은 "2줄에 가깝게, 구체 정보를 담아 충실히" 작성한다.**
  - **개조식(명사형)으로 종결한다.** (`~한다`·`~했다` 등 서술형 금지)
  - **2줄을 목표로 하되 3줄은 금지한다.**
  - 텍스트박스 폭에 맞춰 **자동 줄바꿈**되게 둔다.
- 거버닝 정렬: **왼쪽 정렬**

### 본문 레이아웃 원칙
- **시각 요소(그래프·이미지·도식·차트) = 본문 왼쪽**
- **부연 설명(텍스트·불릿) = 오른쪽**

### 푸터
- 좌: 출처 / 우: 페이지 번호(현재 페이지만, 전체 분량 미표기 — `3` O, `3/6` X)
- "© 회사명 All rights reserved" 류는 표기하지 않는다.
- **출처 표기(간결):** 단일 출처 문서는 모든 슬라이드에 `출처: 문서명(연도)` 형태로 표기.

## 3. 금지 사항 (AI 생성 흔적 / 빈번한 오류)

- ❌ **타이틀 아래 액센트 라인** — AI 생성 흔적의 대표 사례
- ❌ **색 바·사이드 스트라이프·단측 테두리** — 카드 구분은 배경 음영/그림자/아이콘으로 처리
- ❌ **본문 가운데 정렬** — 단락·리스트는 좌측 정렬, 가운데 정렬은 타이틀에만 적용
- ❌ **텍스트 오버플로** — 미수용 시 폰트 축소·분할·박스 확대로 처리
- ❌ **슬라이드 번호(타이틀 옆)** — 페이지는 푸터에만 표기
- ❌ **`fill:none` + `line:{width:0}` 조합** — 빈 외곽 박스는 제거하고, 채움 박스는 `line` 속성 자체를 생략한다.
- ❌ **원문에 없는 데이터 생성** — 원문에 없는 수치/출처 생성 금지
- ❌ **좌/우 2단 컬럼 겹침** — §2 [2단 분할 경계 규칙]의 거터(5.0"~5.15")를 침범하지 않는다.

## 4. 표현 가이드 (콘텐츠 성격 → 표현)

가이드이며, 더 적합한 표현이 있으면 새로 설계한다.

| 콘텐츠 성격 | 표현 |
|---|---|
| 핵심 지표 몇 개 (4h·8%·2h…) | **큰 숫자 콜아웃 카드** (둥근+그림자, 숫자 28~32pt) |
| 현재 → 목표 2시점 비교 | 그룹 막대차트 (현재 회색 / 목표 네이비) |
| 연도별 시계열·성장 | 막대차트 + **CAGR 박스 + 추세 화살표** + 우측 분석 |
| 2축 분류 (영향도×발생가능성) | **버블 매트릭스** (사분면, 우상단 고위험 음영) |
| 항목 나열 (3~5개) | 카드형 (좌 아이콘+키워드 / 우 설명) |
| 단계별 절차 | 프로세스 (원형 번호 배지 + chevron 화살표) |
| 일정·로드맵 | 타임라인 (원형 날짜 배지 + 활동 + 산출물) |
| 비교·대조 (현행 vs 개선) | 2단 비교 (회색 / 연블루 헤더박스) |
| 범위 정의 (포함/제외) | In/Out 박스 (✓ 네이비 / ✕ 회색) |
| 정형 데이터 (다항목×다속성) | 표 (네이비 헤더 + 줄무늬 행) |
| A→B 변환·매핑 | 프로세스 매핑 (chevron 연결) |
| 핵심 메시지 강조 | 인용 강조 박스 (네이비 배경 + 큰 따옴표) |

**차트 슬라이드 필수**: 차트는 왼쪽, 오른쪽에는 차트를 해석하는 **충실한 설명**
(추세·근거·시사점)을 배치한다.

**아이콘**: 모노톤 네이비 심볼을 사용한다(⊞ ◎ ✓ ▤ ⚙ ◉ ⬡ ⚡ 등). 컬러 이모지는 지양한다.

**[도형별 텍스트 안전 패딩]**
- **chevron(화살표)**: 텍스트박스 x를 도형 x보다 **0.28" 들여쓰고** 폭을 그만큼 축소한다.
- **pentagon/arrow 등 비사각 도형**: 형상이 침범하는 방향으로 0.2~0.3" 패딩 확보.
- **원형 배지 안 숫자**: valign·align 모두 center, 폰트가 원 지름의 약 50%를 넘지 않게 한다.
